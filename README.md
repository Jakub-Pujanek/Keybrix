## Auto Updates (electron-updater)

Auto updates are enabled in packaged builds via `electron-updater`.

- Main process service: `src/main/services/auto-update.service.ts`
- Startup trigger: updater starts after the app window is created
- Update source: GitHub Releases configured in `electron-builder.yml` (`publish.provider: github`)
- Release mode for 1.0.0: Draft Release (`releaseType: draft`)

The app expects update metadata (`latest.yml`) and installer artifacts generated by electron-builder and uploaded to the GitHub release.

To temporarily disable updater (for QA/debug), use:

```bash
KEYBRIX_DISABLE_AUTO_UPDATER=1
```

## Release Checklist

For 1.0.0 we publish a GitHub Draft Release without code signing.

Checklist:

1. Build the app with `pnpm build:win` on Windows or `pnpm build:mac` on macOS.
2. Verify that electron-builder generates the updater metadata and install artifacts.
3. Create a Draft Release on GitHub and upload the generated artifacts.
4. Open the packaged app, confirm the updater sees the draft release, and verify the update toast.
5. Click `Install now` only after `update-downloaded` is emitted.
